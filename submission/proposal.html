NeuroGuard: A Memory Isolation & Anomaly Detection Kernel Driver

Device Driver Idea Submission
Authors: Yuval  &  Prajakta
Abstract — We propose NeuroGuard, a Linux kernel character device driver that monitors per-process memory allocation behaviour and detects anomalies using a lightweight, in-kernel Exponential Weighted Moving Average (EWMA) scoring engine. The driver exposes /dev/neuroguard for event streaming, sysfs for runtime tuning, and procfs for human-readable status. When a process's anomaly score breaches a threshold, the driver can alert, throttle (SIGSTOP), or terminate (SIGKILL) the offending process without any user-space polling loop.

1. Motivation

Modern Linux offers no programmable interface that gives a user-space agent fine-grained, real-time telemetry about memory allocation rates. Cgroups enforce hard limits but cannot detect a spike before it becomes fatal. This driver fills that gap for three practical scenarios: detecting heap-spray security attacks, containing runaway ML inference jobs, and notifying operators of memory leaks in production services before they cause OOM kills.

3. Key Technical Components

  • Character device — open, blocking read, write (quick-attach), ioctl, poll
  • Workqueue poller — delayed_work fires every 500 ms; reads RSS via get_mm_rss()
  • EWMA engine — Q16.16 fixed-point (no FPU); score = α·|ΔRSS| + (1−α)·prev
  • kfifo ring — 512-slot lock-free producer; wait-queue for blocking read()
  • sysfs — alpha_pct, threshold, poll_ms tuneable live
  • procfs — /proc/neuroguard/summary table of active sandboxes

4. Event & ioctl Interface

Each anomaly event dequeued via read():

struct neuroguard_event {
  pid_t  pid;
  u64    timestamp_ns;
  u32    anomaly_score;  /* Q16.16 */
  s64    mem_delta_bytes;
  u32    event_type;     /* ALERT / THROTTLE / KILL */
  char   comm[16];
};
ioctl CommandPurpose
NG_ATTACH_PIDStart monitoring a PID
NG_DETACH_PIDStop monitoring a PID
NG_GET_SCORERead current EWMA score
NG_SET_ACTIONSet ALERT / THROTTLE / KILL

5. Demo Plan (April 26)

  • Load module → confirm /dev/neuroguard appears
  • Run neuroguard-ctl monitor in one terminal
  • Launch memory-bomb: allocates 80 MB bursts every 250 ms
  • Attach bomb's PID → watch live ALERT events with rising scores
  • Switch to THROTTLE → bomb stalls; switch to KILL → process exits
  • Live-tune alpha via sysfs; show faster/slower score response
  • Unload module; show clean dmesg teardown

7. Learning Outcomes

This project covers the full Linux driver stack: file_operations, ioctl, user↔kernel data transfer, kfifo, wait queues, workqueues, kobject/sysfs, seq_file/procfs, and mm_struct internals — alongside practical design tradeoffs in polling latency, statistical sensitivity, and safe in-kernel arithmetic.
